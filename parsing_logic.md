# Логика парсинга страховых полисов (Monday_parse)

Этот документ описывает текущую логику работы парсера `api/extract.js`. Парсер обрабатывает PDF-файлы страховых полисов и извлекает ключевые данные.

## Основные типы полисов
На данный момент система различает и поддерживает четыре типа полисов:
1. **Автоцивілка (RCA / ОСЦПВ)** — стандартная внутренняя страховка.
2. **Повний Автозахист** — комплексная внутренняя страховка (формат TAS).
3. **Зелена картка (GREEN_CARD)** — международная страховка.
4. **Travel** — международная комплексная страховка (формат TAS).

Тип полиса определяется автоматически по наличию в тексте ключевых слов:
- *Зелена картка, Green Card, Carte Internationale, КАРТКА МІЖНАРОДНОГО АВТОМОБІЛЬНОГО СТРАХУВАННЯ* → Зелена картка
- *ПОВНИЙ АВТОЗАХИСТ* → Повний Автозахист
- *МІЖНАРОДНИЙ ДОГОВІР КОМПЛЕКСНОГО СТРАХУВАННЯ* → Travel
- Всё остальное → Автоцивілка

В выходном JSON поле `insurance_type` возвращает либо `"Зелена картка"`, либо `"Автоцивілка"`. Это предотвращает выполнение неподходящих регулярных выражений для каждого из типов.

## Как извлекаются данные (Отличия и нюансы)

### 1. Номер полиса (`policy_number`)
- **Автоцивілка:** Ищет стандартные 9-значные номера после слов `Поліс №` или `Акцепт) №`.
- **Зелена картка:** Добавлены специфические форматы номеров зеленой карты, например `UA/078/41812487` или `UA048.41846155`. Скрипт извлекает только значимую числовую часть.

### 2. Цена (`price`)
Копейки (всё, что после точки или запятой) во всех случаях отбрасываются. Нужны только целые числа.
- **Автоцивілка:** Ищет в секции `15 Розмір страхової премії` или рядом со словами `Страховий платіж`.
- **Зелена картка:** Двухэтапный подход:
  1. Сначала ищет заголовок `Розмір страхової премії`.
  2. В пределах 3000 символов от заголовка ищет паттерн `(\d[ \t\d]*\d)(?=[,.]00)` — число перед `,00` или `.00`.
  3. Запасной вариант: ищет просто число рядом со словами `Страхова премія` или `Страховий платіж`.

  **Важный нюанс:** Старый паттерн `((?:\d\s*)+)` вызывал backtrack и мог захватить цифры из даты (например, `3720` из `05.07.2026` вместо цены `16022`). Новый паттерн `(\d[ \t\d]*\d)(?=[,.]00)` требует обязательного наличия `,00` или `.00` перед числом, что исключает захват лишних цифр.

### 3. Имя страхователя (`insured_name`)
- **Автоцивілка:** Регулярные выражения настроены на поиск кириллицы (украинского алфавита). Ищет в блоках "3. Страхувальник" или после слова "Найменування".
- **Зелена картка:** ФИО обычно пишется **латиницей**. Добавлены дополнительные паттерны, которые активируются только для Зеленой карты. Они ищут латинские имена (например, `ANASTASIIA KARIAKINA`) в 3-й секции, а также имеют запасной вариант поиска в 9-й секции ("ПІБ ТА АДРЕСА СТРАХУВАЛЬНИКА").

### 4. Дата начала (`start_date`)
- **Автоцивілка:** Обычно включает время. Скрипт ищет формат вроде `З 00:00 04 січня 2025 р` или `з 00:00 12.05.2025`.
- **Зелена картка:** Несколько вариантов:
  1. Стандартный: `з DD.MM.YYYY` или `по DD.MM.YYYY` (с временем или без).
  2. Формат "Договоруз": `Договоруз[ \t\n]*DD.MM.YYYY` — дата идёт сразу после текста `Договоруз` без пробела (так бывает в PDF-таблицах ПЗУ).
  3. Формат "Договору з": `Договору\s+з\s+DD.MM.YYYY` — с пробелами.

### 4.1 Дата окончания (`end_date`)
- **Автоцивілка:** Ищет паттерн `по DD.MM.YYYY` с временем или без.
- **Зелена картка:** Несколько вариантов:
  1. Стандартный: `по DD.MM.YYYY` или `по 00:00 DD.MM.YYYY`.
  2. Формат "Договорупо": `Договорупо[ \t\n]*DD.MM.YYYY` — дата идёт сразу после текста `Договорупо` без пробела.
  3. Формат "Договору по": `Договору\s+по\s+DD.MM.YYYY` — с пробелами.

### 5. Гос. номер авто (`car_number`)
- **Основа:** Сначала ищет возле явных маркеров вроде `4.2. Реєстраційний номер` или `Номерний знак`.
- **Запасной вариант 1 (Глобальный):** Если номер не найден, скрипт просканирует весь текст документа на наличие автомобильных номеров стандартного украинского формата (вида `АА 1234 ВВ`, `АА1234ВВ` или `12345 АА`). 
- **Запасной вариант 2 (Только для Зеленой карты):** Если предыдущие этапы не дали результата (например, это номер ЕС из 6-7 букв/цифр без пробелов), скрипт сфокусируется строго на блоке `5. ДЕРЖАВНИЙ НОМЕРНИЙ ЗНАК ... 6. КАТЕГОРІЯ`. Он вытянет оттуда слово из заглавных букв и цифр, отфильтровав служебные термины вроде `REGISTRATION`, `ENGINE`, `ПРИ`, `НОМЕР` и т.д.

## 🛠 Руководство по внесению изменений
Если в будущем потребуется поддержка новых бланков:
1. **Не ломайте старое:** Добавляйте новые `fullText.match(...)` через оператор `||` (ИЛИ) к существующим переменным.
2. **Ограничивайте область видимости:** Если новый формат специфичен только для одного типа полиса, оборачивайте его в условие:
   ```javascript
   if (!myVariable && isGreenCard) {
       // Логика только для зеленой карты
   }
   ```
3. **Осторожно с глобальным поиском:** Старайтесь искать значения по ближайшим маркерам-якорям (заголовкам секций), чтобы не вытащить случайные цифры или слова из других частей документа.
