# Monday_parse — Документация проекта

## Что делает этот инструмент

**Monday_parse** — это серверный API на Node.js/Next.js, который принимает PDF-файл страхового полиса в формате base64, извлекает из него ключевые данные и возвращает структурированный JSON.

### Входные данные
HTTP POST-запрос с телом:
```json
{ "pdfData": "<base64-строка PDF>" }
```

### Выходные данные
```json
{
  "success": true,
  "result": "1223|2814919278|281491927",
  "details": {
    "insurance_type": "Зелена картка",
    "price": "1223",
    "ipn": "2814919278",
    "policy_number": "281491927",
    "insured_name": "KARAINAKI OLEKSANDR",
    "start_date": "04.06.2026, 00:00",
    "end_date": "18.06.2026",
    "car_model": "VOLKSWAGEN TOURAN",
    "car_number": "BH0720TH",
    "vin_number": "WVGZZZ1TZAW084872",
    "phone": "380963694232"
  }
}
```

Поле `result` — строка `price|ipn|policy_number` для быстрой вставки в Monday.com.

## Поддерживаемые типы полисов

### 1. Автоцивілка (ОСЦПВ)
- Стандартний внутрішній поліс цивільно-правової відповідальності
- Формати номерів: `EP-XXXXXXXXX`, `611933-XXXX-XXXXXXXXX` (9 цифр в кінці)
- Дата починається з часу: `З 00:00 04 січня 2025 р.`
- Ціна: секція `15 Розмір страхової премії` або `Страховий платіж`
- Ім'я: блок `3. СТРАХУВАЛЬНИК` або `Найменування`

### 2. Зелена картка (Green Card)
- Міжнародне страхування, діє за кордоном
- Формати номерів: `UA/078/41812487`, `UA048.41846155`, `Поліс UA/ 42320944`
- Ім'я може бути латиницею
- Підтримуються формати різних страховиків:
  - Стандартний: секція `найменування Страхувальника`
  - Картковий: секція `ПІБ ТА АДРЕСА СТРАХУВАЛЬНИКА`
  - Княжа: секція `3СТРАХУВАЛЬНИК` → ім'я латиницею на наступному рядку

### 3. Повний Автозахист
- Комплексний договір страхування, надбудова над ОСЦПВ-полісом
- Номер формату: `FO-XXXXXXXX` (наприклад, `FO-03394998`)
- Страхувальник: не зазначений у тілі документу — лише скорочене ФІО у рядку підпису (`ДУДНІК О. А.`)
- Телефон: у секції `Підписано за допомогою одноразового ідентифікатору ... відправленого на номер +380...`
- Ціна: рядок `«Автоцивілка плюс»50 000,00200,00` — другe число (премія)
- Посилається на базовий ОСЦПВ-поліс

### 4. Travel (Мандрівник)
- Міжнародний договір комплексного страхування під час перебування за кордоном
- Номер формату: `611933-4421-XXXXXXX` (наприклад, `611933-4421-1000090`)
- Страховик: Євроінс Україна
- Ім'я: з таблиці `П.І.Б. / Name` (латиниця або кирилиця)
- Телефон: поле `Тел.:`
- Ціна: поле `13. Загальний страховий платіж, грн.`
- Немає авто (car_model, car_number, vin → null)

```javascript

// Пріоритет: Повний Автозахист → Зелена картка → Travel → Автоцивілка
```

## Реалізовані поля та звідки беруться

| Поле | Автоцивілка | Зелена картка | Повний Автозахист | Travel |
|------|-------------|---------------|-------------------|--------|
| `insurance_type` | `"Автоцивілка"` | `"Зелена картка"` | `"Повний Автозахист"` | `"Travel"` |
| `policy_number` | 9-значний номер після `Поліс №` або `Акцепт) №` | `UA/XXX/...` або `UA048....` | `Номер договору FO-XXXXX` | `No 611933-4421-XXXXX від` |
| `price` | `15 Розмір страхової премії` або `Страховий платіж` | `10 Розмір страхової премії` або `Страхова премія` | Рядок `Автоцивілка плюс ... 200,00` (2-га сума) | `13. Загальний страховий платіж` |
| `ipn` | `РНОКПП`, `ЄДРПОУ`, `ІНПП` | те саме | — (ЄДРПОУ = страховик, не беремо) | `РНОКПП` |
| `insured_name` | Секція `3. СТРАХУВАЛЬНИК` або `Найменування` | Секція страхувальника (латиниця або кирилиця), формат Княжа | Рядок підпису перед `Підписано за допомогою` | Таблиця `П.І.Б. / Name` |
| `start_date` | `З ЧЧ:ХХ ДД місяць РРРР р.` або `з ЧЧ:ХХ ДД.ММ.РРРР` | `з ДД.ММ.РРРР` (без часу → `00:00`) | — | — |
| `end_date` | `5.2`, `до ДД.ММ.РРРР`, `по ... р.` | те саме | — | — |
| `car_model` | Секція `4.1. Марка, модель` або `9.2/9.3. Марка/Модель` | те саме | — (авто в ОСЦПВ-полісі) | — (немає авто) |
| `car_number` | `4.2. Реєстраційний номер`, `Номерний знак` | те саме + fallback на секцію `5. ДЕРЖАВНИЙ НОМЕРНИЙ ЗНАК` | — | — |
| `vin_number` | `VIN`, `Номер кузова` | те саме | — | — |
| `phone` | Після слова `Телефон` або `Номер телефону` в секції страхувальника | те саме | Секція підпису `відправленого на номер +380...` | Поле `Тел.:` |

## Файл парсера

`api/extract.js` — єдиний файл з усією логікою. Структура: inline if-else блоки з каскадними `||` для кожного поля. Нові типи додаються через окремі `if (!field && isXxx)` блоки в кінці кожної секції — стара логіка не зачіпається.
